# Content & Collections

Mirador is optimized for IIIF Presentation and Image API resources. It can display digitized books,
manuscripts, artworks, maps, archival material, audio, video, and collections.

## Manifests

A manifest describes a resource and its canvases. Mirador uses manifest data for titles, metadata,
canvas order, available image services, rights, search services, and structural navigation.

## Collections

Collections let users move between related manifests. Mirador demo examples show both loading a
collection and navigating between collections and manifests.

## Static Images

Some IIIF resources use static image files instead of IIIF Image API services. Mirador includes
integration examples for these level 0 or static-image use cases.

## Audio And Video

Mirador supports audio and video content through IIIF Presentation API v3 resources. When a canvas
contains time-based media, the available controls and annotations depend on the resource structure and
viewer features.

## Right-To-Left Viewing

Mirador can display resources that page or flow right-to-left when the IIIF resource provides the
appropriate `viewingDirection` metadata.
