# User Guide

This section is for readers who need to operate Mirador once it is already available on a web page.
It focuses on the viewer concepts you will meet in the interface: resources, windows, canvases,
metadata, navigation, annotations, search, and workspace layouts.

## Core Concepts

### IIIF Resource

Mirador is designed around IIIF resources, especially manifests and collections. A manifest usually
describes one digitized object and its canvases. A collection groups manifests or other collections.

### Window

Each open resource appears in a window. A window can show one canvas, a book-like spread, a gallery,
or a scrolling sequence depending on the resource and available views.

### Workspace

The workspace is the area that contains windows. Mirador supports a structured mosaic workspace for
comparison and an elastic workspace for freer arrangement.

### Sidebar

Window sidebars expose panels such as information, attribution, thumbnails, search results, table of
contents, and annotation lists when the loaded resource supports them.
