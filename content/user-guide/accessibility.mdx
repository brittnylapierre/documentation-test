# Accessibility

Mirador’s project documentation sets WCAG 2.1 AA as the accessibility target for contributors.
For users, that means accessibility should be part of the everyday viewer workflow rather than a
separate polish task.

## What Users Should Expect

- Keyboard-accessible controls where the interface exposes actions.
- Screen-reader text for user-facing controls and relevant ARIA attributes.
- Usable contrast in available themes.
- Logical focus behavior while moving through windows, panels, and menus.

## Manual Checks

Automated tools help, but the Mirador accessibility guidelines also call for manual review. Test with
keyboard navigation, browser accessibility tooling, and a representative screen reader setup when you
evaluate a Mirador viewer.
