or research tool. It covers installation and mounting patterns, configuration bundles, plugins,
theme customization, language customization, and common implementation issues.

## Implementation Checklist

1. Decide whether you need a simple bundled viewer or a build that supports plugins.
2. Create a stable container element with explicit height.
3. Start with one manifest and the smallest useful configuration.
4. Add workspace, window, and sidebar options only after the base viewer works.
5. Test the target manifests, collections, search services, annotations, and A/V resources.
6. Review accessibility, theming, and language requirements before launch.

## Getting Started Track

The getting-started flow gives implementers a practical first-run mount path:

- Compare the hosted UMD build with an npm/yarn application dependency.
- Mount Mirador from HTML, bundled JavaScript, or React.
- Create a stable container element with explicit height.
- Confirm one known manifest renders before adding local customization.

After that, use [Configuration](/install-config/configuration), [Theming & i18n](/install-config/theming-i18n),
and [Plugins](/install-config/plugins) to see what else Mirador can do.
